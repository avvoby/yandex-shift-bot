# Пошаговая инструкция по запуску Telegram-бота
## Для человека без опыта программирования

Эта инструкция поможет:
1. создать бота в Telegram;
2. создать Google-таблицу;
3. настроить доступ к Google Sheets;
4. создать сервер в Yandex Cloud;
5. загрузить код из GitHub;
6. запустить бота.

---

# Часть 1. Создаем Telegram-бота

## Шаг 1. Откройте Telegram
Найдите пользователя:

`@BotFather`

## Шаг 2. Создайте бота
Отправьте команду:

`/newbot`

## Шаг 3. Укажите имя бота
Например:

`Мерчендайзинг Support Bot`

## Шаг 4. Укажите username
Он обязательно должен заканчиваться на `bot`.

Например:

`merch_support_project_bot`

## Шаг 5. Сохраните токен
BotFather пришлет вам токен. Он выглядит примерно так:

`123456789:AAExampleToken`

Сохраните его. Позже вставите в файл `.env`.

---

# Часть 2. Создаем Google-таблицу

## Шаг 1. Откройте Google Sheets
Создайте пустую таблицу.

## Шаг 2. Назовите таблицу
Например:

`Merch Bot Database`

## Шаг 3. Создайте листы
Создайте листы с такими названиями:
- users
- content
- faq
- admins
- broadcasts_log
- support_requests
- settings
- shift_confirmations

## Шаг 4. Заполните таблицу
Откройте файл `docs/google_sheets_template.md` и перенесите оттуда:
- названия колонок;
- обязательные ключи;
- примеры строк.

---

# Часть 3. Настраиваем Google Cloud и Google Sheets API

## Шаг 1. Откройте Google Cloud Console

## Шаг 2. Создайте проект
Нажмите создание нового проекта и назовите его, например:

`merch-bot-project`

## Шаг 3. Включите Google Sheets API
Внутри проекта найдите раздел API и включите:
- Google Sheets API

## Шаг 4. Создайте service account
Создайте сервисный аккаунт.

## Шаг 5. Скачайте JSON-ключ
После создания сервисного аккаунта скачайте JSON-файл.

Назовите его:

`credentials.json`

## Шаг 6. Дайте сервисному аккаунту доступ к таблице
Откройте скачанный JSON-файл и найдите email сервисного аккаунта.  
Он выглядит примерно так:

`bot-access@project-id.iam.gserviceaccount.com`

Теперь:
1. откройте Google-таблицу;
2. нажмите `Поделиться`;
3. вставьте email сервисного аккаунта;
4. дайте роль редактора.

# Часть 4. Подготавливаем GitHub

## Шаг 1. Убедитесь, что код уже загружен в GitHub
В репозитории должны быть:
- `main.py`
- папка `app/`
- папка `docs/`
- `requirements.txt`
- `.env.example`

## Шаг 2. Проверьте, что все файлы сохранены
Особенно:
- `app/handlers/admin.py`
- `app/handlers/shift_poll.py`
- `app/services/sheets.py`

---

# Часть 5. Создаем сервер в Yandex Cloud

## Шаг 1. Откройте Yandex Cloud Console

## Шаг 2. Зайдите в Compute Cloud
Найдите раздел создания виртуальной машины.

## Шаг 3. Создайте VM
Рекомендуемые параметры для первого запуска:
- Ubuntu 22.04
- 2 vCPU
- 2 GB RAM

## Шаг 4. Добавьте SSH-ключ
Если у вас уже есть SSH-ключ — добавьте его.
Если нет — создайте новый.

## Шаг 5. Завершите создание VM
После создания у машины появится внешний IP-адрес.
